\chapter{Requisitos funcionales}
\section{Introducción}
En este capítulo se describirá la funcionalidad de alto nivel del sistema.
\section{Requisitos del sistema en conjunto}
\begin{itemize}
\Requisito{1}{El sistema proporcionará una ayuda al desarrollo de videojuegos.}{Imprescindible}{}
\Requisito{2}{El sistema proporcionará un subsistema gráfico listo para ser utilizado.}{Imprescindible}{}
\Requisito{3}{El sistema proporcionará un subsistema de sonido listo para ser utilizado. En esta revisión no se incluye el desarrollo de dicho sistema.}{No implementado en esta revisión.}{}
\Requisito{4}{El sistema proporcionará un subsistema de entrada listo para ser utilizado. En esta revisión no se incluye el desarrollo de dicho sistema.}{No implementado en esta revisión.}{}
\Requisito{5}{El sistema proporcionará un esqueleto para facilitar la implementación de la lógica del juego.}{Imprescindible}{}
\Requisito{6}{El sistema proporcionará el esqueleto típico de las aplicaciones interactivas.}{Imprescindible}{}
\end{itemize}

\section{Requisitos del subsistema gráfico}
En esta sección nos referiremos al subsistema gráfico como sistema gráfico o motor gráfico, para abreviar.

\begin{itemize}
\Requisito{2.1}{El motor gráfico estará orientado a las 3 dimensiones del espacio.}{Imprescindible}{}
\Requisito{2.2}{El motor gráfico estará organizado en torno al uso de polígonos como su estructura básica de representación.}{Imprescindible}{}
	\begin{itemize}
	\Requisito{2.2.1}{Los polígonos soportarán texturizado e iluminación.}{Imprescindible}{}
	\Requisito{2.2.2}{El sistema proveerá un mecanismo para la importación y carga de grandes grupos de polígonos (mallas).}{Imprescindible}{}
	\Requisito{2.2.3}{El motor gráfico proveerá de una estructura básica para el manejo y procesamiento de mallas de polígonos.}{Imprescindible}{}
	\end{itemize}
\Requisito{2.3}{El motor gráfico deberá ser fácilmente desacoplable de la biblioteca gráfica que se utilice.}{Imprescindible}{}
	\begin{itemize}
	\Requisito{2.3.1}{Inicialmente se utilizará la biblioteca gráfica OpenGL.}{Imprescindible}{}
	\end{itemize}
\Requisito{2.4}{El motor gráfico tendrá disponibles entidades de alto nivel para su utilización.}{Imprescindible}{}
	\begin{itemize}
	\Requisito{2.4.1}{Una entidad de alto nivel será el objeto. Los objetos permitirán manipular elementos de la escena como son cámaras y luces de una manera sencilla.}{Imprescindible}{}
	\Requisito{2.4.2}{Otra entidad de alto nivel será la pantalla. Las escenas agrupan objetos y se pintan en la pantalla, para que el jugador pueda verlas.}{Imprescindible}{}
	\end{itemize}
\Requisito{2.5}{El motor gráfico estará orientado a representar gráficos en tiempo real en la pantalla.}{Imprescindible}{}
\end{itemize}

\section{Requisitos del esqueleto para la implementación de la lógica del juego}
\begin{itemize}
\Requisito{5.1}{El sistema proporcionará un mecanismo para crear y procesar entidades concurrentes dentro del juego.}{Imprescindible}{}
\Requisito{5.2}{El sistema proporcionará un mecanismo para el control eficiente y adecuado del tiempo dentro del juego.}{Alta}{}
\end{itemize}

\section{Requisitos del esqueleto de aplicación interactiva}
\begin{itemize}
\Requisito{6.1}{El sistema proporcionará un bucle principal.}{Imprescindible}{}
\Requisito{6.2}{El sistema estará estructurado según el modelo de bucle principal típico de aplicación interactiva: 
	\begin{enumerate}
	\item Leer entrada
	\item Procesar información
	\item Representar resultados
	\item Ir a 1
	\end{enumerate} \ }{Imprescindible}{}
	
\end{itemize}
